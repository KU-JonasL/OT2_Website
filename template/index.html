Online Download of Opentrons Script for the Alberdi lab


Welcome! On this website, The Alberdi Lab download customised version of the opentrons scripts we use. The webiste runs on Render using the Flask Python package running the Github repository KU-JonasL/OT2_Website created by Jonas Greve Lauritsen for the Alberdi Group.

Data submission


In the following section you will have to select what protocol you wish to download, and input what what your run is going to be. The inputs will be integrated into the python script. The python script(s) is renamed after user, protocol, and date for easier tracking. Along with each script will be a Standard Operational Procedure (SOP) to guide the user, and the needed custom labware. Below the submission form there is a explanation of each submission. All fields are required to fill out, with the exception of a csv file must be uploaded for the library protocol option.

Data Submission explanation


The following paragraph is an expanded explanation about what each User Input field is expecting.


  1. Protocol. The protocol list 4 options the user can select from; "Extraction","Library", "qPCR", and "IndexPCR". The Alberdi group has divided their work into 4 sections, however each entail different steps. a. Extraction is a total nucleic extraction, b. Library is Covaris Fragmentation preparation, library building and library purification of extracted samples, c. qPCR is qPCR setup of purified libraries, and d. is Index PCR and following purification of the samples. For even more details of the exact methods check out: [Link] .
  2. User. This option allow the user to ID their downloaded content to them. In the Alberdi group we often use RUN-specific IDs, however you can name it whatever suits your workflow.
  3. Number of Samples. Here the user select the number of samples (1-96) they have in their run. The number of samples will be overriding by the csv file uploaded data during the protocols related to the Covaris fragmentation and Library Building to ensure parity. OBS: will be changed later to verify if the sample number and csv is equal to avoid unfortunate misalignments.
  4. Input- and outputformat. The Input- and outputformats give the user the option to obtain the product from the protocols in a specific container. Each protocol has its specific formats, given how the protocol works. The output of 1 protocol is always a option of the next protocols in line. In the SOPs there will be a material list of specific labware - remember some suppliers might have slightly different dimension between labware options, and it is as such always recommended to check the compatability.
  5. Csv file. The Covaris Fragmentation preparation and Library Building protocols require you, the user, to upload a csv file. The Covaris Fragmentation preparation use the information to normalise samples concentrations (as wished), and the Library Building use it to give either 20 or 10 µM adaptor. In case you are in doubt of the data format, there is a Excel template to for you to download an insert your information in. This ensure the column naming is correct. The web application will attempt to convert the data file into English comma separation and numbers.
